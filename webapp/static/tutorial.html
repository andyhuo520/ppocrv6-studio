1 Why this Studio over official tools

Baidu's PP-OCRv6 was released on June 11, 2026. The official distribution channels are functional but bare:

Official PaddleOCR SDKOfficial HF DemoThis Studio
Install footprint~1.2 GB (Paddle)None, hosted~80 MB
Desktop UICLI onlySingle imageFull GUI
Batch queueDIY scriptingNoBuilt-in
Persistent historyNoNoSQLite, searchable
Parameter tuningCode kwargsNoUI sliders, hot reload
PrivacyLocalUploads to HF127.0.0.1 only
Hardware accelMKL / CUDAServer-sideCoreML / ANE
REST APIDIYNoFastAPI

The official Python pipeline is more capable for layout-heavy documents (table structure, formula recognition). This Studio focuses on giving you a polished local UX for the text detection + recognition pipeline.

2 Prerequisites

OSmacOS 12 Monterey or later (Apple Silicon for CoreML acceleration)
Python3.10 or later, 3.12 recommended
Disk~80 MB total (Python deps + ~6 MB model weights)
NetworkOne-time download of model weights, otherwise fully offline
NOTE
Intel Macs also work but fall back to CPU execution provider (~3-5× slower than CoreML on M-series).

4 Install dependencies

Five packages, ~50 MB total, no PaddlePaddle.

# Create a venv (or reuse existing)
python3 -m venv .venv
source .venv/bin/activate

# Install everything
pip install onnxruntime pillow numpy fastapi "uvicorn[standard]" python-multipart

Verify:

pip list | grep -E "onnxruntime|fastapi|uvicorn"
# onnxruntime  1.26.0
# fastapi      0.137.x
# uvicorn      0.49.x

5 Get the model weights

Skip this if you already have the ppocrv6_onnx/ folder populated. Otherwise download from Hugging Face:

pip install huggingface_hub

python3 - <<'EOF'
from huggingface_hub import hf_hub_download
import shutil, pathlib
base = pathlib.Path("ppocrv6_onnx")
(base/"det").mkdir(parents=True, exist_ok=True)
(base/"rec").mkdir(parents=True, exist_ok=True)
shutil.copy(hf_hub_download("PaddlePaddle/PP-OCRv6_tiny_det_onnx","inference.onnx"),
            base/"det"/"inference.onnx")
shutil.copy(hf_hub_download("PaddlePaddle/PP-OCRv6_tiny_rec_onnx","inference.onnx"),
            base/"rec"/"inference.onnx")
shutil.copy(hf_hub_download("PaddlePaddle/PP-OCRv6_tiny_rec_onnx","char_dict.json"),
            base/"rec"/"char_dict.json")
print("done")
EOF
CHINA
In mainland China, mirror via export HF_ENDPOINT=https://hf-mirror.com before running the snippet.

6 Start the Studio

cd ocr_benchmark_v6
source .venv/bin/activate     # if not already
python3 webapp/server.py

You'll see:

Loading models...
Ready · backend: CoreMLExecutionProvider
INFO:  Uvicorn running on http://127.0.0.1:8765

Open http://localhost:8765 in any browser.

TIP
Look at the bottom-left status indicator. Green dot + CoreML means hardware acceleration is active. CPU means falling back to pure CPU.

7 Using the Studio

Recognize view

  • Drop images onto the left dropzone, or click to browse. Multi-file selection supported.
  • Paste a screenshot anywhere with ⌘V.
  • The queue shows each item with thumbnail, filename, status badge and progress bar.
  • Click Run all to process the queue sequentially. Click any item to view its result on the right.
  • Right panel has three tabs: Text output (plain), Visualization (annotated image), Lines (per-box coordinates table).
  • Footer shows detect / recognize / total milliseconds, box count and active backend.

History view

  • Every recognition is persisted to SQLite under webapp/data/history.db.
  • Searchable by filename or recognized text.
  • Click any card to open the detail modal with the annotated image and full text. Delete individual records or clear all.

Settings view

  • Execution provider: Auto / CoreML / CPU. Changing this reloads the model.
  • Thread count: CPU threads for parallel ops.
  • Detection tuning: binarization threshold, box confidence, unclip ratio, max edge.
  • Save to history toggle to opt out of persistence for sensitive workflows.

8 REST API

The Studio is also a backend for integration. All endpoints on http://localhost:8765.

GET/infobackend, available providers, current config
GET/healthsimple liveness probe
POST/ocrmultipart upload, returns lines + annotated
GET/history?q=&limit=50paginated, searchable
GET/history/{id}full record
DELETE/history/{id}remove single record
POST/history/clearwipe all history
GET/settingsread current config
PUT/settingsupdate config, reloads model

Curl example

curl -X POST http://localhost:8765/ocr \
  -F "file=@my_doc.jpg" | jq .text

Python example

import requests
r = requests.post("http://localhost:8765/ocr",
                  files={"file": open("doc.jpg","rb")})
data = r.json()
print(data["text"])
print(f'{data["n_boxes"]} boxes in {data["total_ms"]}ms')

9 Parameter tuning cheatsheet

ParameterDefaultTry lowerTry higher
Binarization threshold0.20Detect faint text, more noiseSkip uncertain regions
Box confidence0.40Keep marginal boxesDrop low-confidence boxes
Unclip ratio1.40Tighter crops, may clip charactersLarger crops, may merge lines
Max edge960Faster, may miss small textSlower, finer detection

Tip: for screenshots of dense documents, push Max edge to 1280 and lower Binarization threshold to 0.15. For handwriting, bump Unclip ratio to 1.6.

10 FAQ

Recognition is slow, single image takes 5+ seconds

Check the bottom-left backend label. If it says CPU, CoreML is unavailable on your machine. Intel Macs will always fall back to CPU.

Multi-column documents read in wrong order

This is a structural limitation of the detect+recognize pipeline, not a bug. The model knows where text is but doesn't analyze layout. For multi-column documents, use PaddleOCR-VL or GLM-OCR (cloud VLMs).

Want to disable history for privacy

Settings → Data & Storage → flip "Save to history" off. Or pass ?save=false when calling the API directly.

Port 8765 already in use

Edit the last line of webapp/server.py and change port=8765.

Stop the server

Press Ctrl+C in the terminal running server.py.

Run as a launch daemon

Wrap the command in a launchd plist under ~/Library/LaunchAgents/ if you want auto-start at login. Save the plist and run launchctl load.
