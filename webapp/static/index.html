这是什么

PP-OCRv6 本地工作台，一款完全离线的桌面级 OCR 工具，基于百度开源的 PP-OCRv6 Tiny 模型 (1.5 MB) 与 ONNX Runtime。 支持中英文文字识别、表格重建、批量队列、历史档案。 数据全程不出本机。

极小安装体积

~80 MB 包含全部依赖与模型权重，无需安装 PaddlePaddle 框架。

硬件加速

Apple Silicon 自动启用 CoreML / 神经引擎，单页 OCR 通常 200-500 ms。

表格重建

基于几何聚类自动还原表格结构，支持 CSV / Markdown / HTML 多格式导出。

隐私优先

所有图片和文字仅在 127.0.0.1 流转，零云端调用，零遥测。

与官方方案对比

百度 2026 年 6 月 11 日发布 PP-OCRv6。官方目前没有桌面级 UI，分发渠道如下：

官方 Python SDK 官方 HF Demo 本工作台
安装体积 ~1.2 GB (含 Paddle) 无 (云端托管) ~80 MB
桌面 UI 仅 CLI / Python 网页, 单张 完整 GUI
批量队列 需自行编写脚本 不支持 内置
历史档案 无 无 SQLite 可搜索
参数调优 需改代码传参 无 UI 滑杆, 热重载
表格识别 需切换 PP-Structure 无 几何聚类内置
表格导出 需手写脚本 无 CSV/MD/HTML
隐私 本地 上传至 HF 云 仅本机
硬件加速 Paddle MKL / CUDA 服务端 CoreML / ANE
HTTP API 集成 需自行实现 无 FastAPI /ocr

官方仍领先的地方

坦率说明，官方 PaddleOCR 完整流水线支持 Medium 模型 (34.5 MB)、版面分析、 完整的表格结构识别 (PP-Structure / SLANet)、公式识别等能力。 本工作台目前只内置 Tiny 模型 + 几何聚类表格重建， 适合纯文本与规则表格场景。复杂多列文档建议改用 PaddleOCR-VL 或 GLM-OCR。

模型版权

PP-OCRv6 模型权重由百度发布，Apache 2.0 协议。 ONNX 版本托管在 huggingface.co/PaddlePaddle。 源码: github.com/PaddlePaddle/PaddleOCR。

本地接口

所有接口均在 http://127.0.0.1:8765：

GET /info · 引擎与配置信息
POST /ocr · 上传图片返回识别结果 + 表格
GET /history · 历史记录分页
GET /history/{id}/export?fmt=csv|xlsx|md|html|txt · 导出指定记录
PUT /settings · 更新参数 (自动重载模型)

详细安装与集成示例见 部署教程。
